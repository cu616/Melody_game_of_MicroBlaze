# 2026-06-08 谱面文本化与 VGA 双缓冲优化

## 用户要求

- 优化铺面的输入方式，便于玩家/开发者自己修改谱面。
- 优化铺面，使铺面节奏更容易和当前音乐演示对上。
- 优化 VGA 输出方式，减少偶发白色花屏/闪点。

## 本次修改

### MicroBlaze 谱面输入

- 新增 `charts/canon.mbchart`、`charts/faded.mbchart`、`charts/aphasia.mbchart`。
- 新增 `scripts/generate_mb_charts.py`，把文本谱面生成到 SDK 头文件：
  - `Mini_IO.sdk/SeriesIODacSaw/src/generated_charts.h`
- `rhythm_game.c` 不再手写三首歌的 `Song0/Song1/Song2` 数组，而是包含生成头文件。
- 当前三首歌每首 14 个判定点，总计 42 个点，控制在 MicroBlaze 本地 BRAM 可链接范围内。

文本格式：

```text
time_ms lane [hold length_ms]
```

示例：

```text
1200 L
7868 L hold 1200
```

其中 `L/C/R` 分别对应 BTNL(P17)、BTNC(N17)、BTNR(M17)。

改谱流程：

```powershell
& 'C:\Users\lbc\.cache\codex-runtimes\codex-primary-runtime\dependencies\python\python.exe' scripts\generate_mb_charts.py
cmd /c "call D:\Xilinx\SDK\2018.3\settings64.bat && make all"
```

第二条命令需要在 `Mini_IO.sdk/SeriesIODacSaw/Debug` 下执行。只改谱面时通常只需要重编并下载 ELF，不需要重新生成 bitstream。

### VGA 输出优化

- `rhythm_game.c` 的 VGA 状态包不再使用 `mb_an_status[7:5] == 000`，而是使用：
  - `001xxxxx`：写入 shadow frame packet。
  - `010xxxxx + 0xA5`：提交当前 frame。
- `rhythm_video_audio.v` 新增 shadow buffer：
  - `mb_note_tracks_shadow`
  - `mb_hold_tracks_shadow`
  - `mb_button_tracks_shadow`
  - `mb_vga_commit_pending`
- RTL 在 VGA 帧末尾一次性把 shadow buffer 复制到显示用 buffer，避免 MicroBlaze 写包写到一半时被 VGA 扫描读到。
- MicroBlaze VGA 刷新从每 1ms 调整为约每 16ms 一次，减轻 GPIO 包传输压力。

### 节奏调整

- 从实际嵌入播放的 MIDI 文件读取 note-on 时间，再挑选稀疏关键点写入 `.mbchart`。
- 为避免第一颗音开局直接贴到判定线，新增 `AUDIO_START_DELAY_MS = 2200`，游戏开始约 2.2 秒后才开始向 VS1003B 喂 MIDI。
- `.mbchart` 中的时间按“实际 MIDI 事件时间 + 2200ms 预读”填写。
- 这次优先保证课堂演示可打性和内存可放下；如果后续扩展外部存储/更大 BRAM，可把每首 14 点提升到更密的 30-80 点。

## 注意

- 本次 VGA 优化解决的是 MicroBlaze/RTL 显示数据跨帧写入导致的撕裂类花屏。
- 如果仍有少量随机白点，需要继续检查 VGA 线缆、显示器采样兼容性、VGA 电阻网络/约束时序，以及像素时钟约束。
