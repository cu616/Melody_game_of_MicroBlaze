# 按键与拨码开关作用分析

本文记录对当前本地 `Mini_IO` 工程中板载按键和拨码开关用途的只读分析。分析只读取代码并新增本文档，不修改 RTL、约束、SDK 或其他工程文件，以避免影响并行工作的其他 AI。

## 主要参考文件

- `Mini_IO.srcs/sources_1/new/rhythm_video_audio.v`
- `Mini_IO.srcs/sources_1/bd/design_mb/hdl/design_mb_wrapper.v`
- `Mini_IO.srcs/constrs_1/new/adda.xdc`
- `Mini_IO.sdk/SeriesIODacSaw/src/rhythm_game.c`

当前 `design_mb_wrapper.v` 将板载拨码开关和按钮同时接给 MicroBlaze block design 与 `rhythm_video_audio`：

```text
dip_switches_16bits_tri_i -> design_mb + rhythm_video_audio.switches
push_buttons_5bits_tri_i  -> design_mb + rhythm_video_audio.buttons
reset                     -> design_mb + rhythm_video_audio.reset
```

也就是说，硬件 RTL 和 SDK 软件都能看到同一批物理输入，但它们对这些输入的解释并不完全一致。

## 物理按键位序

当前 `adda.xdc` 注释给出的 Nexys4 DDR 板载按钮位序为：

| 输入位 | 物理按钮 | 约束注释中的封装引脚 |
| --- | --- | --- |
| `push_buttons_5bits_tri_i[0]` | `BTNC` | `N17` |
| `push_buttons_5bits_tri_i[1]` | `BTNU` | `M18` |
| `push_buttons_5bits_tri_i[2]` | `BTNL` | `P17` |
| `push_buttons_5bits_tri_i[3]` | `BTNR` | `M17` |
| `push_buttons_5bits_tri_i[4]` | `BTND` | `P18` |

`rhythm_video_audio.v` 中有明确重映射：

```verilog
wire [2:0] lane_button_raw = {buttons[3], buttons[0], buttons[2]};
```

因此硬件音游三轨按键实际为：

| 游戏轨道 | `game_buttons` 位 | 来源按钮位 | 物理按钮 |
| --- | --- | --- | --- |
| 左轨 | `game_buttons[0]` | `buttons[2]` | `BTNL` |
| 中轨 | `game_buttons[1]` | `buttons[0]` | `BTNC` |
| 右轨 | `game_buttons[2]` | `buttons[3]` | `BTNR` |

硬件 RTL 还用 `BTNU/BTND` 控制音量：

```verilog
volume_button_meta <= {buttons[1], buttons[4]};
```

| 功能 | 来源按钮位 | 物理按钮 | 作用 |
| --- | --- | --- | --- |
| 音量加 | `buttons[1]` | `BTNU` | `volume_level` 最高加到 7 |
| 音量减 | `buttons[4]` | `BTND` | `volume_level` 最低减到 0 |

音量默认值为 `4`。每次按键边沿触发后有约 `8,000,000` 个 100 MHz 周期的冷却时间，约 80 ms，避免一次按下连续跳多级。

## RTL 中拨码开关的作用

`rhythm_video_audio.v` 中当前直接使用 `switches[0]` 到 `switches[5]`，并使用 `switches[14]`/`switches[15]` 作为测试与暂停控制。

| 拨码位 | RTL 信号 | 当前作用 |
| --- | --- | --- |
| `SW0` / `switches[0]` | `canon_mode` / `audio_enabled` | 单独打开时选择 Canon 风格音频；同时也使能 J8 音频和硬件游戏核心 |
| `SW1` / `switches[1]` | `edm_mode` / `audio_enabled` | 打开时选择 EDM/Fade 风格音频；优先于 SW0 选择游戏曲目 1 |
| `SW2` / `switches[2]` | `vs1003_demo_enabled` | 打开 VS1003B 播放器，并让 LED 低 8 位显示 VS1003B debug |
| `SW3..SW5` / `switches[5:3]` | `game_speed` | 普通模式下选择硬件音游下落/滚动速度；VS1003B 校音模式下不参与音高选择 |
| `SW14` / `switches[14]` | `vs1003_pitch_test` | 与 `SW2` 同时打开时进入标准音校准模式，用于判断 VS1003B 是否整体走音 |
| `SW15` / `switches[15]` | `game_paused` | 暂停硬件音游 |
| `SW6..SW13` | 未发现 RTL 直接使用 | 当前硬件 RTL 中未找到明确功能 |

模式细节：

```verilog
wire canon_mode = switches[0] && !switches[1];
wire edm_mode = switches[1];
wire audio_enabled = switches[0] || switches[1];
wire vs1003_demo_enabled = switches[2];
wire vs1003_pitch_test = switches[14];
wire [1:0] game_song = edm_mode ? 2'd1 : 2'd0;
wire [1:0] vs1003_song_select = vs1003_pitch_test ? 2'd2 : game_song;
wire [2:0] game_speed = switches[5:3];
```

因此：

| SW1 | SW0 | RTL 音频/游戏选择 |
| --- | --- | --- |
| 0 | 0 | J8 音频关闭，硬件游戏核心被 `enable=0` 复位/停用 |
| 0 | 1 | Canon 模式，游戏曲目 0 |
| 1 | 0 | EDM/Fade 模式，游戏曲目 1 |
| 1 | 1 | EDM/Fade 模式优先，游戏曲目 1 |

`SW2` 与 `SW0/SW1` 相对独立：`SW2` 使能 VS1003B 播放器，但硬件音游核心仍由 `SW0/SW1` 的 `audio_enabled` 控制。若 `SW14=1`，`SW2` 连续循环播放 C 大调校音音阶；若 `SW14=0`，`SW2` 按 `SW0/SW1` 播放 Canon/Faded MIDI。当前 MIDI 资源已按实测听感整体下移 1 个半音，用来抵消该 VS1003B 模块约高半音的输出偏差。

VS1003B C 大调校音音阶为：

| 顺序 | 音名 | 理论频率 |
| ---: | --- | ---: |
| 1 | `C4` | `261.63 Hz` |
| 2 | `D4` | `293.66 Hz` |
| 3 | `E4` | `329.63 Hz` |
| 4 | `F4` | `349.23 Hz` |
| 5 | `G4` | `392.00 Hz` |
| 6 | `A4` | `440.00 Hz` |
| 7 | `B4` | `493.88 Hz` |
| 8 | `C5` | `523.25 Hz` |

用手机调音器测这组连续音阶：如果所有音整体偏高/偏低，优先怀疑 VS1003B 时钟或初始化；如果音阶基本准确而歌曲不像，优先修改 MIDI 乐谱。

## 硬件音游速度选择

`switches[5:3]` 送入 `rhythm_game_core.speed_select`，其速度表如下：

| `SW5 SW4 SW3` | 速度含义 |
| --- | --- |
| `000` | 0.75x |
| `001` | 1.00x |
| `010` | 1.25x |
| `011` | 1.50x |
| `100` | 1.75x |
| `101` | 2.00x |
| `110` | 2.50x |
| `111` | 3.00x |

注意位序是 `switches[5:3]`，也就是 `SW5` 是三位速度值的最高位，`SW3` 是最低位。

## 硬件音游中的按钮判定

`rhythm_game_core` 对按钮做同步和上升沿检测：

```verilog
button_edges <= button_sync & ~button_prev;
```

点按音符判定：

- 左轨按下 `BTNL`，判断 `lane0`。
- 中轨按下 `BTNC`，判断 `lane1`。
- 右轨按下 `BTNR`，判断 `lane2`。

判定窗口逻辑：

- `lane[26..28]` 命中为好判定，`judgement=2`，加 100 分。
- `lane[24..25]` 或 `lane[29..31]` 命中为较差判定，`judgement=1`，加 40 分。
- 没命中时该次按键不加分，`judgement` 会走较差/空判定路径。
- 音符滚到 `lane[31]` 仍未处理时为 miss，`judgement=0`，combo 清零。

长按音符：

- 如果 `hold_laneX[31]` 且对应按钮仍按住，则每个滚动周期加少量分。
- 如果 `hold_laneX[31]` 但对应按钮未按住，则视为 miss。

## LED、RGB 和七段数码管反馈

在 `vs1003_demo_enabled=0` 时，`diag_led` 显示硬件音游状态：

| LED 位 | 显示内容 |
| --- | --- |
| `LED[2:0]` | 三轨命中窗口 `game_hit_window` |
| `LED[7:5]` | 当前同步后的三轨按钮状态 `game_buttons` |
| `LED[11:10]` | 当前游戏曲目 `game_song` |
| `LED[15:12]` | 当前判定 `game_judgement` |

当 `SW2=1` 时，`diag_led = {8'h00, vs1003_debug}`，LED 改为显示 VS1003B demo 的 debug 状态。

RGB LED 判定颜色：

| `game_judgement` | 含义 | RGB 显示 |
| --- | --- | --- |
| `2` | good | 绿色 |
| `1` | bad | 蓝色 |
| `0` | miss | 红色 |
| 其他 | 空闲或无判定 | 有按键时白色，否则熄灭 |

七段数码管由硬件 `rhythm_sevenseg` 显示：

- 低位显示 score 的 BCD 数字。
- 高位部分显示 combo 和 judgement。

## SDK 软件中的按钮和拨码开关逻辑

`Mini_IO.sdk/SeriesIODacSaw/src/rhythm_game.c` 里也有一套软件音游控制逻辑，但当前文件顶部：

```c
#define USB_UART_AUDIO_TEST 1
```

因此 `main()` 会进入 `RunUsbUartAudioTest()` 并无限循环发送 UART 音频测试数据，后面的软件音游按钮/开关逻辑不会实际运行。

如果以后把 `USB_UART_AUDIO_TEST` 改成 `0`，软件逻辑会解释按钮为：

| 软件宏 | 位值 | 软件认为的功能 |
| --- | --- | --- |
| `BTN_U` | `0x01` | 开始游戏 |
| `BTN_D` | `0x04` | 游戏中暂停；暂停中继续 |
| `BTN_L` | `0x02` | 左轨判定 |
| `BTN_C` | `0x10` | 中轨判定 |
| `BTN_R` | `0x08` | 右轨判定 |

软件中的拨码开关逻辑：

| 拨码位 | 软件逻辑作用 |
| --- | --- |
| `SW0..SW1` | 游戏时间步进增量：`GameTimeMs += 1 + (sw & 0x0003)`，即 1 到 4 ms/tick |
| `SW2` | 在 `StartGame()` 时选择 `SongIndex = (sw >> 2) & 1` |
| `SW3..SW15` | 当前软件逻辑未发现直接使用 |

重要差异：软件宏的按钮位序与 `adda.xdc` 注释和 RTL 重映射并不一致。例如 `adda.xdc` 注释中 `buttons[0]` 是 `BTNC`，但软件把 `0x01` 当成 `BTN_U`。不过在当前 `USB_UART_AUDIO_TEST=1` 的状态下，这套软件按钮逻辑不会运行。

## Reset 的作用

`rhythm_video_audio.v` 中：

```verilog
wire reset_active = ~reset;
```

因此传入 RTL 的 `reset` 是低有效；按下或拉低 reset 会复位 VGA/音频/游戏/音量状态。文档交接中也提醒不要随意改这个极性，除非 block design 的 reset 极性同步调整。

## 当前状态小结

当前真正主导屏幕、J8 音频、硬件音游判定的是 `rhythm_video_audio.v`：

- `SW0/SW1` 选择并启用 J8 音频和硬件音游。
- `SW2` 启用 VS1003B demo/debug 显示。
- `SW3..SW5` 控制硬件音游速度。
- `BTNL/BTNC/BTNR` 对应左/中/右三轨。
- `BTNU/BTND` 控制 J8 音频音量。

SDK 中也有软件音游逻辑，但当前被 UART 音频测试宏短路；除非修改 `USB_UART_AUDIO_TEST`，否则软件中的按钮和拨码开关解释不会参与游戏运行。
