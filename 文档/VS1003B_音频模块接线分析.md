# VS1003B 音频模块接线分析

本文记录对本工程接入 VS1003B 音频输出模块的只读分析。分析参考了本地文档区的 `Digital_Logic_FPGA_final_hw` 工程，尤其是其中的 `MP3.v` 与 `VGA_TOP.v`，并对照了当前 `Mini_IO` 工程已有的 MicroBlaze、AXI Quad SPI、约束文件与 J8 PWM 音频实现。

## 参考工程中的 VS1003B 使用方式

`Digital_Logic_FPGA_final_hw/MP3.v` 中的 `mp3board` 模块使用 VS1003B 作为 MP3 解码输出设备。它不是接 FPGA 的模拟音频输出脚，而是通过 SPI 类接口把控制命令和音频数据送入 VS1003B，由 VS1003B 自己完成解码和模拟音频输出。

参考模块端口含义如下：

| 参考代码端口 | VS1003B 模块信号 | 方向 | 作用 |
| --- | --- | --- | --- |
| `XDCS` | `XDCS` | FPGA 输出 | SDI 数据接口片选，低有效 |
| `XCS` | `XCS` | FPGA 输出 | SCI 控制接口片选，低有效 |
| `DREQ` | `DREQ` | FPGA 输入 | 数据请求，高电平表示可以继续传输 |
| `SCK` | `SCLK` | FPGA 输出 | SPI 时钟 |
| `SI` | `MOSI` | FPGA 输出 | FPGA 写入 VS1003B |
| `SO` | `MISO` | FPGA 输入 | VS1003B 回读数据 |
| `XRESET` | `XRST` | FPGA 输出 | 硬复位，低有效 |

参考代码的基本流程：

1. 硬复位 `XRST/XRESET`。
2. 等待 `DREQ` 有效。
3. 通过 `XCS` 选中 SCI 控制接口，写入软复位、音量、音调、时钟等寄存器。
4. 播放时通过 `XDCS` 选中 SDI 数据接口，按 `DREQ` 流控持续送 MP3 数据。
5. `MISO/SO` 在参考实现里声明了，但核心播放路径几乎不依赖读回。

## 手头模块 9 个引脚的连接需求

用户手头 VS1003B 模块从上到下引脚为：

```text
XDCS XCS DREQ SCLK MOSI MISO XRST GND 5V
```

建议连接关系：

| 模块引脚 | 作用 | FPGA 方向 | 是否需要连接 |
| --- | --- | --- | --- |
| `XDCS` | 数据接口 SDI 片选，低有效 | 输出 | 必须 |
| `XCS` | 控制接口 SCI 片选，低有效 | 输出 | 必须 |
| `DREQ` | VS1003B 请求数据，高电平可传 | 输入 | 必须 |
| `SCLK` | SPI 时钟 | 输出 | 必须 |
| `MOSI` | FPGA 写入 VS1003B | 输出 | 必须 |
| `MISO` | VS1003B 回读 FPGA | 输入 | 建议连接 |
| `XRST` | 硬复位，低有效 | 输出 | 强烈建议连接 |
| `GND` | 共地 | 电源地 | 必须 |
| `5V` | 模块供电 | 电源 | 必须，但需确认模块电平转换/稳压 |

概念上，VS1003B 应该使用：

```text
共享 SPI: SCLK + MOSI + MISO
两个片选: XCS + XDCS
两个控制: DREQ + XRST
电源: GND + 5V
```

## 当前 Mini_IO 工程中的相关现状

当前工程已有以下外部 SPI 相关端口，约束在 `Mini_IO.srcs/constrs_1/new/adda.xdc`：

| 当前端口 | FPGA 封装引脚 |
| --- | --- |
| `SS1[0]` | `C17` |
| `MISO` | `D18` |
| `SCLK1` | `G17` |
| `SS0[0]` | `D14` |
| `MOSI` | `F16` |
| `SCLK0` | `H14` |

但当前 Block Design 里 SPI 结构不是一条完整普通 SPI 总线：

| 当前信号 | 来源 |
| --- | --- |
| `MOSI` | `axi_quad_spi_0/io0_o` |
| `SCLK0` | `axi_quad_spi_0/sck_o` |
| `SS0` | `axi_quad_spi_0/ss_o` |
| `MISO` | `axi_quad_spi_1/io1_i` |
| `SCLK1` | `axi_quad_spi_1/sck_o` |
| `SS1` | `axi_quad_spi_1/ss_o` |

这意味着不能简单认为当前 `MOSI/MISO/SCLK0/SCLK1/SS0/SS1` 已经天然适配 VS1003B。VS1003B 更适合一条共享 `SCLK/MOSI/MISO`，外加 `XCS/XDCS` 两个独立片选。

## 建议的实际接线方向

如果后续另一个实现者要把 VS1003B 接入当前工程，建议目标连接抽象为：

```text
模块 MOSI  -> FPGA MOSI
模块 MISO  -> FPGA MISO
模块 SCLK  -> FPGA SCLK0 或新建共享 SCLK
模块 XCS   -> FPGA SS0[0] 或独立 GPIO 输出
模块 XDCS  -> FPGA SS1[0] 或独立 GPIO 输出
模块 DREQ  -> 新增 FPGA GPIO 输入
模块 XRST  -> 新增 FPGA GPIO 输出
模块 GND   -> Nexys4 DDR GND
模块 5V    -> Nexys4 DDR 5V 供电
```

更稳妥的软件/硬件架构是：

1. 只保留一条 SPI 总线给 VS1003B：`SCLK/MOSI/MISO`。
2. 用两个 GPIO 或 SPI slave select 分别控制 `XCS` 和 `XDCS`。
3. 用一个 GPIO 输入读取 `DREQ`。
4. 用一个 GPIO 输出控制 `XRST`。
5. 初始化阶段用 `XCS` 写 SCI 寄存器。
6. 播放阶段用 `XDCS` 按 `DREQ` 流控喂数据。

## 电源和电平注意事项

VS1003B 芯片逻辑通常是 3.3V，Nexys4 DDR 的用户 IO 也是 `LVCMOS33`。用户模块提供 `5V` 供电脚，通常说明模块板上可能带 3.3V 稳压，但这需要用模块资料或实测确认。

注意：

- `5V` 可以作为模块供电前提是模块明确支持 5V 输入。
- FPGA IO 不能直接承受 5V 信号。
- 所有信号线必须确认是 3.3V 逻辑兼容。
- `GND` 必须和 Nexys4 DDR 共地，否则 SPI/DREQ/XRST 信号没有可靠参考。

## 与当前 J8 PWM 音频的关系

当前 `Mini_IO` 工程已有板载 J8 单声道 PWM 音频输出，通过 `AUD_PWM` 和 `AUD_SD` 实现。这和 VS1003B 是两套不同音频路径：

| 音频路径 | 特点 |
| --- | --- |
| J8 PWM | FPGA 直接产生简单音频，音质有限，占用 `AUD_PWM/AUD_SD` |
| VS1003B | FPGA 通过 SPI 喂 MP3/音频数据，由外部模块解码输出，适合真实音乐 |

接入 VS1003B 不需要连接 `AUD_PWM/AUD_SD`。如果目标是更像真实歌曲的音游背景音乐，VS1003B 路线比 J8 PWM 更合适。

## 目前不建议立即做的事

为避免影响并行代码修改，本文只记录分析，不修改工程。后续真正接入时，不建议只靠飞线接上当前 `SS0/SS1/SCLK0/SCLK1` 就期望可用，因为当前 SPI 资源拆成两个 AXI Quad SPI，和 VS1003B 的“一条 SPI + 双片选”模型不完全一致。

推荐后续实现者先决定：

1. 使用纯 RTL `mp3board` 风格直接驱动 VS1003B；
2. 或用 MicroBlaze + AXI Quad SPI + GPIO 软件驱动 VS1003B。

两条路线都可行，但接线抽象应保持一致：共享 SPI，总线外再单独处理 `XCS/XDCS/DREQ/XRST`。
